# 献血管理系统测试报告

## 1. 测试目标

本次测试用于确认系统核心业务规则、权限边界、数据一致性和报表数据处理符合需求，并验证 MongoDB 日志写入达到课程要求的 10000 条日志、50 并发规模。

测试重点：

- 密码哈希、登录成功与失败日志。
- 注册输入校验和重复用户拒绝。
- 用户、分类、库存、评论和订单的业务规则。
- 审批用血申请时的库存扣减。
- MySQL 与 MongoDB 数据的统计和联合展示。
- 数据删除后统计界面不使用过期行。
- 压力测试写入数量、并发数和数据清理。

## 3. 测试策略

### 3.1 业务测试

服务层测试使用可控 DAO 替身，直接验证输入、输出、权限和业务调用结果。这类测试不依赖外部数据库，适合在每次改动后快速回归。

### 3.2 结构与回归测试

DAO 结构测试检查课程要求的 DAO 类、继承关系和事务方法是否存在。UI 回归测试检查删除数据后的统计行处理和 MySQL 日期时间转换。

### 3.3 压力测试

压力测试默认不随常规测试执行，需要显式设置 `stressTests=true`。这样日常回归测试不强制依赖 MongoDB，验收时又可以单独重复完整压力测试。

## 6. 业务规则测试

### 6.1 分类管理

- 分类名称在保存前去除首尾空格。
- 分类不能选择自身为父分类。
- 分类不能选择自身的子分类为父分类。
- 存在子分类时不能删除。
- 分类已被库存使用时不能删除。

以上用例全部通过。

### 6.2 库存与 MongoDB 详情

- 新增库存时同时保存 MySQL 主记录和 MongoDB 详情。
- 更新库存时同步更新主记录和详情。
- 删除库存后删除对应 MongoDB 详情。
- 主记录不存在时拒绝单独保存详情。
- 可按 MySQL `item_id` 查询 MongoDB 详情。

以上用例全部通过。

### 6.3 评论和评分

- 只能对存在的库存批次发表评论。
- 评论内容不能为空。
- 评分范围为 1 至 5。
- 管理员可删除任意评论，普通用户只能删除自己的评论。
- 用户评分报表只统计当前用户的评论。

以上用例全部通过。

### 6.4 用户和档案

- 普通用户只能维护自己的档案。
- 只有超级管理员可修改其他用户的角色。
- 不能删除当前登录用户。
- 不能删除超级管理员。
- 用户有关联业务记录时，删除失败并返回明确提示。

以上用例全部通过。

### 6.5 用血申请和库存扣减

- 创建申请前检查库存存在、状态可用且数量充足。
- 停用库存不能创建申请。
- 申请数量大于可用库存时拒绝创建。
- 普通用户只能查询自己的申请。
- 普通用户只能修改或删除自己的待审批申请。
- 审批通过时调用事务方法扣减库存。
- 取消申请只更新状态，不扣减库存。

以上用例全部通过。
