# 献血管理系统用户手册

## 1. 系统简介

献血管理系统用于统一管理血液库存批次、用血申请、用户档案、评论评分、统计报表和系统日志。

系统提供两种角色：

- 管理员：维护用户、分类和库存，处理用血申请，查看报表和日志。
- 普通用户：查询库存，提交用血申请，维护个人档案，发表评论并查看个人统计。

## 2. 运行前准备

### 2.1 环境要求

- JDK 17 或更高兼容版本。
- Maven 3.9 或可兼容版本。
- MySQL 8.0。
- MongoDB 5.0。
- 可显示桌面窗口的操作系统。

### 2.2 启动 Docker 数据库

项目默认使用以下容器：

- MySQL：`hzcu-mysql`
- MongoDB：`hzcu-mongo`

启动命令：

```bash
docker start hzcu-mysql hzcu-mongo
```

查看容器状态：

```bash
docker ps --filter name=hzcu-
```

等待两个容器显示 `healthy` 后启动系统。

### 2.3 数据库配置

连接配置文件：

```text
src/main/resources/db.properties
```

默认连接目标：

| 数据库 | 地址 | 数据库名 |
| --- | --- | --- |
| MySQL | `localhost:3306` | `hzcu_mysql` |
| MongoDB | `localhost:27017` | `hzcu_mongo` |

如本机的端口、账号或密码不同，需要先修改配置文件。

## 3. 初始化数据

### 3.1 MySQL 初始化顺序

1. `mysql_clear.sql`
2. `mysql_schema.sql`
3. `mysql_views.sql`
4. `mysql_procedures.sql`
5. `mysql_triggers.sql`
6. `mysql_init_data.sql`

执行命令：

```bash
docker exec -i hzcu-mysql mysql -uroot -p123456 < src/main/resources/sql/mysql_clear.sql
docker exec -i hzcu-mysql mysql -uroot -p123456 < src/main/resources/sql/mysql_schema.sql
docker exec -i hzcu-mysql mysql -uroot -p123456 < src/main/resources/sql/mysql_views.sql
docker exec -i hzcu-mysql mysql -uroot -p123456 < src/main/resources/sql/mysql_procedures.sql
docker exec -i hzcu-mysql mysql -uroot -p123456 < src/main/resources/sql/mysql_triggers.sql
docker exec -i hzcu-mysql mysql -uroot -p123456 < src/main/resources/sql/mysql_init_data.sql
```

### 3.2 MongoDB 初始化顺序

1. `mongodb_clear.js`
2. `mongodb_init.js`
3. `mongodb_init_data.js`

执行命令：

```bash
docker exec -i hzcu-mongo mongosh \
  -u root -p 123456 --authenticationDatabase admin \
  < src/main/resources/sql/mongodb_clear.js
docker exec -i hzcu-mongo mongosh \
  -u root -p 123456 --authenticationDatabase admin \
  < src/main/resources/sql/mongodb_init.js
docker exec -i hzcu-mongo mongosh \
  -u root -p 123456 --authenticationDatabase admin \
  < src/main/resources/sql/mongodb_init_data.js
```

`clear` 脚本会清除已有项目数据。重新初始化前应确认当前数据无需保留。

## 4. 启动系统

先编译项目：

```bash
mvn compile
```

在 IDE 中运行：

```text
com.blooddonation.Main
```

也可使用 Maven 执行主类：

```bash
mvn exec:java -Dexec.mainClass=com.blooddonation.Main
```

系统启动后显示“献血管理系统登录”窗口。

## 5. 演示账号

| --- | --- | --- | --- |
| 管理员 | `admin` | `admin123` | 启用 |

## 6. 登录与登出

### 6.1 登录

1. 在“用户名”输入框填写用户名。
2. 在“密码”输入框填写密码。
3. 点击“登录”，或按回车键。
4. 登录成功后进入主界面。

常见提示：

| 提示 | 处理方式 |
| --- | --- |
| 请输入用户名和密码 | 补全两个输入项 |
| 用户名或密码错误 | 检查用户名和密码 |
| 账号已停用 | 联系管理员启用账号 |
| 数据库连接失败 | 检查 MySQL、MongoDB 容器和配置文件 |

### 6.2 登出

1. 点击左下角“登出”。
2. 在确认提示中选择确认。
3. 系统关闭主界面并返回登录页。

## 7. 注册账号

1. 在登录页点击“注册账号”。
2. 填写用户名、密码和邮箱。
3. 手机号可以不填；如填写，必须为 11 位数字。
4. 点击“注册”。
5. 注册成功后，返回登录页使用新账号登录。

注册账号的默认角色为普通用户。

输入要求：

- 用户名不能为空，且不能与已有用户名重复。
- 密码不能为空。
- 邮箱不能为空，且必须符合常见邮箱格式。
- 手机号填写时必须为 11 位数字。

## 9. 业务数据

“业务数据”显示血液库存批次，主要字段包括库存批次、分类、血型、数量、状态、评论数、行为数和订单数。

### 9.1 查询库存

1. 点击“业务数据”。
2. 使用状态、血型或分类筛选数据。
3. 双击某一行查看批次详情。

详情页包含批次基本信息、详细描述、评论和相关业务操作。

### 9.2 新增库存

仅管理员可操作。

1. 点击“新增”。
2. 填写库存名称。
3. 选择分类和血型。
4. 输入大于 0 的库存数量。
5. 填写描述。
6. 点击“保存”。

基本信息保存到 MySQL，扩展描述和血型信息保存到 MongoDB。

### 9.3 编辑库存

1. 双击需要修改的库存批次。
2. 在详情页点击“编辑”。
3. 修改名称、分类、数量、状态、血型或描述。
4. 点击“保存”。

库存状态包括“可用”和“停用”。停用批次不能用于新的用血申请。

### 9.4 删除库存

仅管理员可操作。库存批次已被用血申请引用时，数据库会拒绝删除。

## 10. 推荐批次

“推荐批次”根据当前用户的申请记录、行为记录、批次热度和评分推荐可用库存。

推荐理由可能包括：

- 你最近申请过同类库存。
- 近期关注度较高。
- 可用库存推荐。

申请推荐批次：

1. 在推荐列表中选择一行。
2. 点击“申请”。
3. 输入用血数量。
4. 确认保存。

## 11. 订单记录

订单记录对应用血申请和处理记录。

| 状态 | 含义 |
| --- | --- |
| 待审批 | 申请已创建，库存尚未扣减 |
| 已完成 | 申请已审批，库存已扣减 |
| 已取消 | 申请已取消，不扣减库存 |

### 11.1 创建申请

1. 点击“订单记录”。
2. 点击“新增”，或从库存详情、推荐列表点击“申请”。
3. 选择可用库存批次。
4. 输入大于 0 且不超过可用库存的数量。
5. 点击“申请”或“创建”。

### 11.2 修改或删除自己的申请

普通用户只能修改或删除自己的待审批申请。已完成或已取消记录不能作为待审批申请编辑。

### 11.3 审批申请

仅管理员可操作。

1. 双击待审批记录。
2. 将状态更新为“已完成”。
3. 系统在同一 MySQL 事务中更新申请状态并扣减库存。

库存不足、库存已停用或申请不是待审批状态时，系统拒绝审批。

### 11.4 取消申请

管理员可将待审批记录更新为“已取消”。取消不扣减库存。

## 12. 评论与评分

1. 在“业务数据”中双击库存批次。
2. 进入评论区域。
3. 点击“发表评论”。
4. 填写评论内容。
5. 选择 1 至 5 分。
6. 选择或填写标签。
7. 点击“发布”。

普通用户只能删除自己的评论；管理员可删除任意评论。

## 13. 用户管理与个人档案

### 13.1 管理员维护用户

管理员可在“用户管理”查看用户的角色、状态、联系方式和实名档案。

可维护内容：

- 邮箱。
- 手机号。
- 真实姓名。
- 证件号。
- 地址。
- 备注。
- 账号状态。
- 用户角色。

权限限制：

- 只有超级管理员可修改其他用户的角色。
- 不能删除当前登录账号。
- 不能删除超级管理员。
- 用户已有关联业务记录时，删除会被拒绝。

### 13.2 普通用户维护档案

1. 点击“我的档案”。
2. 点击“编辑”。
3. 修改邮箱、手机号、姓名、证件号、地址或备注。
4. 点击“保存”。

普通用户不能修改自己的角色和账号状态。

## 14. 分类管理

仅管理员可使用。分类支持父子层级，用于表示血型、血液成分和业务类型。

### 14.1 新增分类

1. 点击“新增”。
2. 填写分类名称。
3. 根据需要选择父分类。
4. 点击“保存”。

### 14.2 编辑分类

分类不能将自己或自己的子分类设置为父分类，以免形成循环层级。

### 14.3 删除分类

以下情况不能删除：

- 分类下还有子分类。
- 分类已被库存批次使用。

## 15. 统计报表

### 15.1 管理员报表

| 页签 | 内容 |
| --- | --- |
| 操作热度 | 按库存批次统计行为次数 |
| 评论评分 | 按批次统计评论数和平均评分 |
| 月度报表 | 按月份统计申请数量和完成用量 |
| 订单状态 | 统计待审批、已完成和已取消数量 |
| 分类分布 | 按库存分类统计申请 |
| 血型分布 | 按血型统计申请 |
| 月度趋势 | 展示月度申请数量变化 |
| 我的评分 | 显示当前账号发表的评分统计 |

月度报表可使用左右按钮切换月份。

### 15.2 普通用户报表

普通用户可查看自己的订单状态、分类分布、血型分布、月度趋势和评分统计。

## 16. 系统日志

仅管理员可从导航进入“系统日志”。

| 页签 | 内容 |
| --- | --- |
| 行为日志 | 查看用户的库存查询、申请、评论等操作 |
| 登录日志 | 查看登录成功、失败和警告信息 |
| 操作审计 | 按日志类型和级别统计数量与最近时间 |

可使用操作类型、批次、级别和日志类型筛选表格。

## 17. 建议演示流程

答辩现场建议按以下顺序演示：

1. 使用 `admin` 登录，展示管理员导航。
2. 新增或编辑一条库存批次。
3. 查看批次的 MySQL 主数据和 MongoDB 详情。
4. 使用 `user01` 登录，创建一条用血申请并发表评论。
5. 切换回 `admin`，将该申请审批为已完成。
6. 展示库存数量扣减。
7. 展示月度报表、评论评分和操作热度。
8. 展示登录日志和操作审计。

## 18. 常见问题

### 18.1 登录时提示数据库连接失败

检查：

1. `hzcu-mysql` 和 `hzcu-mongo` 是否处于运行且健康状态。
2. `3306` 和 `27017` 端口是否被其他服务占用。
3. `db.properties` 中的账号、密码和数据库名是否正确。
4. 初始化脚本是否已执行。

### 18.2 库存或评论信息显示不完整

详情、评论和日志保存在 MongoDB。确认 MongoDB 容器已启动，并且已执行 MongoDB 初始化和演示数据脚本。

### 18.3 无法删除用户、分类或库存

数据可能已被其他业务记录关联。这是数据一致性保护，应根据系统提示先处理子分类或关联记录。

### 18.4 月度报表无数据

切换到存在申请记录的月份。初始演示数据包含 2026 年 6 月、7 月和 8 月的记录。

## 19. 数据安全提示

- 演示账号只用于课程环境。
- 不要在演示数据中录入真实身份证号、手机号或医疗隐私信息。
- 对需要保留的数据执行清理脚本前，应先备份。
- 对外部署时应更换数据库默认密码，并不再将真实凭据保存在项目文件中。

## 20. 退出系统

使用“登出”返回登录页。如需完全结束系统，关闭登录窗口。日常演示结束后可保留数据库容器运行；如需停止，执行：

```bash
docker stop hzcu-mysql hzcu-mongo
```
